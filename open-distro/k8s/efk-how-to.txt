opendistro_security.compliance.salt to be a random string of 16 chars length identical on all nodes

1. Force to re-allocate failed shards:
   curl -X POST https://user:pwd@elasticsearch:9200/_cluster/reroute?retry_failed=true

2. Get indexes:
   curl -X GET https://user:pwd@elasticsearch:9200/_cat/indices
   curl -X POST https://user:pwd@elasticsearch:9200/_cat/indices/index-name?pretty
   curl -X GET https://user:pwd@elasticsearch:9200/_cat/indices/_all?v

3. Get verbose info on shards allocations and problems
   curl -X GET https://elasticsearch:9200/_cluster/allocation/explain?pretty

4. Set nuber of replicas
   curl -XPUT "localhost:9200/<INDEX_NAME>/_settings?pretty" -H 'Content-Type: application/json' -d' { "number_of_replicas": 2 }'

5. Enable shard allocation
   curl -X PUT "localhost:9200/_cluster/settings?pretty" -H 'Content-Type: application/json' -d'
{
    "transient" : {
        "cluster.routing.allocation.enable" : "all"
    }
}
